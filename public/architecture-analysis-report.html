🧠 4. CHALLENGES LOGIQUE MÉTIER
🎯 QUESTION 1: Que devient la file à 12h ?
Réponse technique: Les tickets doivent être conservés en base avec status 'WAITING'. À 14h, reprise automatique du premier ticket en attente.
🎯 QUESTION 2: Ticket en cours à 11:59 ?
Réponse technique: Ne jamais fermer brutalement. Le guichet reste ouvert jusqu'à la fin du ticket actuel. Fermeture douce après completion.
🎯 QUESTION 3: Client en attente pendant pause ?
Réponse technique: Notification WebSocket: "Service temporairement suspendu - Reprise à 14h". Affichage sur écran d'attente.
🎯 QUESTION 4: Agent oublie d'ouvrir ?
Réponse technique: Si horaire actif et guichet fermé: ouverture automatique via scheduler. Notification agent/admin.

🏗️ 8. RECOMMANDATIONS ARCHITECTURE
🎯 Recommandation 1: Approche Graduelle
Implémenter par étapes pour éviter les régressions. Commencer par les améliorations non-breaking.
🎯 Recommandation 2: Events-First
Utiliser les events pour toute logique métier. Éviter le code couplé.
🎯 Recommandation 3: Testing
Tests unitaires pour chaque nouvelle méthode. Tests d'intégration pour les workflows complets.
🎯 Recommandation 4: Monitoring
Ajouter des logs pour toutes les actions automatiques. Monitoring des performances scheduler.
